# Release Notes

## Update Policy

When a phase is completed:

1. Update phase documentation in `docs/phases/phaseX/`.
2. Publish summary in `docs/reports/phaseX_report.md`.
3. Update archive pointers in `docs/archives/phaseX_archive_index.md`.
4. Refresh relevant sections of this handbook.
