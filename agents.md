# Git Working Rules

- This folder is a self-contained git repository.
- Work directly on `main` unless a different branch is explicitly requested.
- Push completed work to `origin/main`.
- Use one-line high-signal commit messages with conventional prefixes:
  - `feat:`
  - `fix:`
  - `docs:`
  - `chore:`
  - `refactor:`
  - `test:`
- Keep commit messages concise and professional.
- Never add `Co-authored-by`, `Made-with: Cursor`, AI attribution, or any note saying the commit was generated by an assistant.
- Commit messages must be a single line title only (no trailers/footers).
- Enforce trailer stripping with local hook: `.git/hooks/commit-msg` removes `Made-with: Cursor`.
- Before pushing, verify with:
  - `git log -20 --grep="Made-with: Cursor" --pretty=oneline`
  - `git log -1 --pretty=full`
- Prefer multiple small logical commits over one large commit.
- Commit implementation, docs, and experiment assets separately when possible.
- Do not commit files that are intentionally ignored by `.gitignore`.
- Avoid destructive git commands unless explicitly requested.
